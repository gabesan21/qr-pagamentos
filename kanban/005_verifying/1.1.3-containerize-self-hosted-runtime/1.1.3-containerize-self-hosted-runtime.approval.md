2. `1.1.3-containerize-self-hosted-runtime.plan.md:22,28,50` and `subtasks/1.1.3-containerize-self-hosted-runtime.g02-compose-startup.md:9-11` combine fixed UID 1000 processes with mode-0600 host file secrets but never establish source-file ownership/readability by UID 1000. Compose file-backed secrets are bind mounts, so mode alone is insufficient and a clean host whose invoking UID is not 1000 can fail before bootstrap/migrate/app. Specify a portable preflight/ownership contract, test readability inside every consuming service as its configured UID without printing values, and make criterion 2 observe it.
3. `1.1.3-containerize-self-hosted-runtime.plan.md:24,28,53,55,82` does not make URL construction and redaction adversarially testable. Require reserved-character passwords that prove SQL escaping plus URL percent-encoding, force at least one real connection failure per relevant wrapper, and assert exact sentinel/URL absence from stderr and service logs.
4. `1.1.3-containerize-self-hosted-runtime.plan.md:54` gives only named PASS labels for restart behavior. It does not state an observable assertion that an app restart/recreation leaves the completed bootstrap and migrate container IDs, exit codes, and execution timestamps unchanged, so `PASS app-restart` can be emitted after accidentally rerunning one-shot jobs. Add those before/after observations (and the expected `unless-stopped` versus `restart: "no"` behavior) to the lifecycle pass contract.
5. `1.1.3-containerize-self-hosted-runtime.plan.md:56` is vacuous for part of criterion 8. The harness requires a clean committed worktree before archiving (`subtasks/1.1.3-containerize-self-hosted-runtime.g03-clean-clone-harness.md:9-11`), so bare `git diff --check` checks an empty working-tree diff rather than the task range; the listed commands also do not necessarily prove the asserted README health/rollback/cleanup and critical-human content. Check the committed task range (for example, the merge-base with `origin/develop`) and add executable documentation/DOX assertions with explicit PASS output.
