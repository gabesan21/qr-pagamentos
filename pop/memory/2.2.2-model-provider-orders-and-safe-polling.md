# 2.2.2-model-provider-orders-and-safe-polling — Model provider orders and safe polling

Delivered durable owner-bound Nautt quote and order state on `develop`. Forward-only Prisma models and PostgreSQL constraints preserve exact decimal text, composite ownership, lifecycle/status validity, provider identifiers, ambiguity, and versioned reconciliation. Order creation atomically claims a quote and persists a `CREATING` attempt before the single provider POST; every outcome becomes `CREATED` or `INDETERMINATE`, with no automatic second POST. A known provider UUID may be recovered only through one explicit authoritative owner-key read, while unknown ambiguity remains manual. Polling is an injected one-order operation with no invented scheduler or cadence, and the shared owner/provider/state/version CAS permits active progress while preventing stale reads or any regression of a final state.
