# M-2.1-self-updating-safe-migrations — Self-updating safe migrations

- **Delivery:** Added a bare self-update command that fast-forwards the tracked branch, binds update work to one revision, validates migration policy offline, runs a fresh migrate container before app promotion, and preserves the existing database volume and Nautt key on failure.
- **Areas changed:** installer/update and its fixture, migration-policy verifier and contracts, migration container and update harness, Docker/Compose contracts, operator documentation, administrative spec, and root/Prisma DOX.
- **Verification:** installer, migration-policy, database and container static contracts, syntax, lint, and scope checks passed. Docker runtime checks were blocked by Docker socket access; `pnpm test` and typecheck retain unrelated pre-existing failures. The human explicitly accepted the remaining baseline co-mutation regression-test concern as overengineering.
- **Durable decisions:** destructive migration operations remain inexpressible in the future manifest language; the legacy baseline inventory has an independent digest anchor; update serialization and revision rechecks prevent checkout drift; migration success precedes target app promotion.
- **Deviation:** the stricter regression that replays a matching altered-SQL/hash baseline pair was intentionally not added by explicit human decision.
