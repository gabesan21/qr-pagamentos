# 1.1.3-containerize-self-hosted-runtime — Containerize the self-hosted runtime

Delivered pinned multi-stage production images and Compose orchestration for PostgreSQL, role bootstrap, one-shot Prisma migrations, runtime database preflight, and the non-root Next.js application. Only the application is published, bound to loopback on the operator-selected `APP_PORT`; PostgreSQL and operational jobs remain internal. File-backed secrets preserve distinct administrator, migrator, and runtime credentials without committed values or privilege escalation.

Added clean-clone container contracts and privilege-free `install/` procedures for hosts where Docker Engine and Compose v2 are already available to the invoking user. The Docker build context excludes PoP worktrees, preventing nested generated-code typecheck failures. Automated container, database, quality, installer, digest, documentation, DOX, and cleanup checks passed, followed by the user's successful integrated Docker installation test. The delivery was merged directly to `main` at `73edf3f` by explicit human instruction; the final handoff is `542e85e`, also synchronized to `develop` by the yolo critic.
