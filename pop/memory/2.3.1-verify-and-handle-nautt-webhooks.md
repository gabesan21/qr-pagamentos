# 2.3.1-verify-and-handle-nautt-webhooks — Verify and handle Nautt webhooks

Delivered the central `POST /api/nautt/webhooks` trust boundary on `develop`. The sessionless route consumes no more than 256 KiB through a byte-preserving stream, verifies `X-Nautt-Signature: sha256=<hex>` against every active owner's encrypted one-time secret before parsing, accepts exactly one constant-time match, and returns only empty no-store protocol responses. A forward-only PostgreSQL ledger uniquely deduplicates provider delivery UUIDs, records every authenticated attempt, serializes claims, holds a fenced 16-second processing lease, reclaims expired work, and prevents stale workers from finalizing.

Webhook payload status is never trusted for payment mutation. Known active orders perform at most one authoritative owner-key `GET /orders/{uuid}` and reuse the shared owner/provider/version CAS; replays, unknown orders, terminal orders, forged input, and live concurrent claims perform zero provider reads. Verification proved malformed-signature cleanup, bounded-stream cancellation, one-worker races, retry outcomes, UUID mismatch isolation, notification-status exclusion, ACL, and a deterministic 12.6-second 1,000-owner path under the 14.5-second budget. `pnpm db:test`, database contracts, `pnpm check` (485 passed, 3 skipped), build, PoP, and two independent critical reviews passed. Task branch was merged locally and pushed to `develop` as `0f4be6d` without a task PR.
