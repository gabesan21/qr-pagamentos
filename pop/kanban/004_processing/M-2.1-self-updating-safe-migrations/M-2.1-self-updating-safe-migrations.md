# M-2.1-self-updating-safe-migrations — Self-updating deployment with safe migrations

- **Origin:** [[MODIFICATIONS|M-2]].
- **Plan:** [[M-2.1-self-updating-safe-migrations.plan]] · **Approval:** [[M-2.1-self-updating-safe-migrations.approval]] · **Verification:** [[M-2.1-self-updating-safe-migrations.verify]]

## What

Make `install/update.sh` pull the latest fast-forward revision from its tracked remote branch, update the existing Docker deployment, and execute all pending migrations without requiring backup or previous-release arguments. Establish and enforce the invariant that project migrations never destroy or discard user data.

## Why

The current command delegates source updating to the operator, requires administrative metadata that does not itself protect data, and does not make the project's non-destructive migration policy executable.
