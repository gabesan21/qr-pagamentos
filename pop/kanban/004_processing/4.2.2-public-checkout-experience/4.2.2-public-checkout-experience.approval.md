# Approval — [[4.2.2-public-checkout-experience]]

## Decision brief

- **Delivery:** a bilingual, self-hosted, sessionless checkout page whose fields come only from the trusted policy, calls only the existing checkout POST, and displays only the returned redacted PIX/capability facts.
- **New contract requiring this gate:** approve `/pay/[identifier]`, its exact `{ product, checkoutPolicy }` server projection, and `POST /api/payment-links/[identifier]/checkout/status`; the successful status body and its eight-state vocabulary are closed in the lifecycle spec, while every syntactically valid capability loser is empty `404`/`no-store`.
- **Safety:** an intent freezes one key and payload; checkout is never auto-retried. Polling is single-flight, abortable, visibility-aware, terminal/unavailable-stopping, and retries only the status read.
- **UI:** all policy, loading, validation, submit, QR/copy, pending/indeterminate, terminal, expiry/unavailable, and recovery states are dictionary-backed in `pt-BR`/`en`, built from the current PIX-ledger primitives with keyboard and live-region semantics.
- **Proof:** focused resolver/route/state/i18n tests, static accessibility/token checks, `pnpm check`, design-system evidence verification, and diff check. The user explicitly skips browser runtime and screenshot evidence; Docker and disposable DB checks remain out of scope.
- **Plan:** [[4.2.2-public-checkout-experience.plan]] — follow for ownership, API/redaction contract, and acceptance criteria.
