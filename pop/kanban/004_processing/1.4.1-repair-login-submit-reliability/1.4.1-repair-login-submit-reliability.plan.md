## Route

First extend the existing clean-clone container harness with a deterministic `login` scenario that POSTs form-encoded credentials to the seeded production app. Run that probe before changing authentication behavior and save only sanitized status, response-header, application-log, and safe database-state evidence in the task diagnosis artifact. The probe must exercise invalid credentials first, then the generated seeded password, inspect redirects and cookie attributes, and confirm the negotiated locale was persisted.

Use the observed exception to make the smallest root-cause repair at its owning boundary. Add a focused regression that fails for the observed defect, plus route/service coverage proving infrastructure exceptions are not disguised as invalid credentials. Do not add a broad catch that converts database/configuration failure to an authentication denial, and do not redesign the login page in this task.

Commit the candidate before the clean-clone acceptance run, execute the focused tests and full gates, then record the diagnosis, chosen fix, rejected alternatives, and redaction check. Review every affected DOX contract; update one only if ownership, structure, inputs, outputs, or editing rules changed.

## Abort conditions

## Red-team

- **Attack that failed against the plan:** mocked unit tests could pass while production login still fails. The mandatory clean-clone production POST is a `re-run` gate and cannot be replaced by mocks.
- **Attack that passed → patch:** a generic `catch` could make the 500 disappear by falsely reporting invalid credentials, masking an outage and satisfying a shallow status check. The plan now requires an injected infrastructure-failure test, no invalid-credential redirect/cookie for that case, and diagnosis before behavior changes.
- **Attack that passed → patch:** capturing Compose logs could leak generated passwords or database URLs. The diagnosis is restricted to sanitized fields, the harness redaction sentinel remains mandatory, and any detected secret aborts the task and invalidates the artifact.
