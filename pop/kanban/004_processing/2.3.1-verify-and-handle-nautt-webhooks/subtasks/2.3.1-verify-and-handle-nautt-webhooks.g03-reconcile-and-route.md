# Group g03 — Reconcile and route — [[2.3.1-verify-and-handle-nautt-webhooks]]

- **Order:** 3
- **Depends on:** g01, g02
- **Description:** Connect durable decisions to one owner-bound provider read and the central unlocalized route.

## Movements

- [x] (agent) Extend `ProviderOrderStore` with exact owner+provider UUID lookup and factor polling/recovery/webhook through the existing reconciliation primitive; active or known-UUID-indeterminate rows may GET, final/unknown rows may not.
  - Expected: one matched owner API key is decrypted, one 10-second GET is attempted, returned UUID is checked, and the same owner/provider/state/version CAS applies; stale/final results are no-op.
  - If it fails: factoring changes polling/recovery behavior → retain their characterized public methods and add the smallest shared internal primitive protected by existing and new tests.
- [x] (agent) Implement intake orchestration around the delivery claim: deterministic rejected/ignored/duplicate/busy/retryable/processed outcomes, attempt evidence, no transaction across network, and durable finalize before `204`.
  - Expected: forged/malformed/unknown/final/duplicate paths follow the plan matrix; an actionable request performs ≤1 GET; provider or finalize failure returns `503` and can be reclaimed on a later request.
  - If it fails: order updates succeed but delivery finalize fails → return `503`; the next claim re-reads current order and safely finalizes via CAS/no-op, never compensates or retries internally.
- [x] (agent) Add `src/app/api/nautt/webhooks/route.ts` and tests; route applies the declared-length precheck, uses the one bounded streaming reader, delegates one byte-exact `Buffer`, and returns empty generic `204/400/401/413/503` responses with `Cache-Control: no-store`.
  - Expected: the route is unlocalized and sessionless; chunked/no-length oversize cancels early at the first `>256 KiB` observation with zero crypto/parse/write/fetch; accepted chunks are HMACed unchanged; valid work returns within the injected 15-second budget without leaking errors.
  - If it fails: framework code parses, fully buffers, or consumes twice → keep all body access in the route adapter and pass one bounded `Buffer` to the service; no middleware/body helper may consume it first.
- [x] (agent) Run focused signature, intake, route, store, and reconciliation suites and capture network/decryption/write counts for every outcome.
  - Expected: all focused tests pass with 0 GET for forged/duplicate/unknown/final and ≤1 for actionable retry.
  - If it fails: any case exceeds its economic count → fix the branch before aggregate gates; do not accept retries hidden in adapters.
