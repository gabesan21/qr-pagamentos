## Blocking findings

1. **Blocking — authentication side-effect gate is not observable.** `src/integrations/nautt/webhook-intake.test.ts:24-38` does not spy on parsing, delivery persistence, API-key decryption, or provider fetch, although criterion 2 names each as a zero-call requirement. Add injected/observable boundaries or focused tests that fail if any of those operations occur for missing, malformed, forged, or ambiguous signatures.
2. **Blocking — HTTP and busy outcome matrix is incomplete.** `src/app/api/nautt/webhooks/route.test.ts:27-55` covers only `204` and `413`, and `src/integrations/nautt/webhook-intake.test.ts` has no live-lease/busy case. Add route assertions for empty, `Cache-Control: no-store` `400/401/503` responses (including thrown intake/read failures as applicable) and an intake case proving `busy -> 503` with zero reconciliation/provider work.
3. **Blocking — authoritative-read trust boundary lacks direct negative tests.** `src/integrations/nautt/provider-order-reconciliation.test.ts:96-116` proves the happy and ignored paths but never returns a different UUID from the provider, and the intake test does not prove notification status is excluded from reconciliation state. Add cases proving mismatched returned UUID fails without CAS and arbitrary notification status cannot influence the authoritative state write.
