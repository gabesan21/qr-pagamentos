## Blocking findings

### B1 — Static lockups depend on an unbundled system font

`scripts/generate-brand-assets.mjs:31-38` emits each public lockup with a live
`<text font-family="'IBM Plex Sans',sans-serif">` element. These files are
explicitly the positive/reversed exports for external or static contexts, but
an SVG loaded as an image is isolated from the application's local
`@font-face`; on a machine without IBM Plex installed it renders the generic
fallback and changes the wordmark metrics/appearance. The manifest still calls
the bytes deterministic and the design contract says every lockup uses the
licensed IBM Plex Sans family.

**Required correction:** make the static wordmark self-contained and
deterministic (for example, repository-owned reviewed outlines derived from the
licensed local font), or stop shipping lockup files whose rendering cannot meet
the stated family contract. Add an inspection/check proving there is no
external or host-font dependency.

### B2 — The manifest checker does not enforce the closed four-ID family

`src/brand/geometry.ts:17-22` is copied directly into
`manifest.identities` at `scripts/generate-brand-assets.mjs:155-159`.
`validateGeneratedContract()` at lines 192-228 validates geometry, SVG shape,
and ICO entries but never compares identity IDs with the required closed set or
requires one composition/export mapping per ID. Adding an unknown ID to
`brandIdentityIds`, regenerating, and running `brand:check` therefore succeeds.
That contradicts the plan's closed-ID/unknown-rejection contract and leaves a
future family expansion invisible to the fail-closed guard.

**Required correction:** independently assert the exact four stable identity
IDs and their required composition/export mapping, with a mutation-focused test
that proves unknown, missing, and duplicate IDs fail.
