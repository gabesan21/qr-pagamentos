## Blocking findings

### B1 — accepted orders need not end in a unique stable key

- **Evidence:** `src/data-directory/server/directory-page.ts:58-71` validates
  only order length, distinct field names, and identifier syntax. An order with
  only a non-unique field such as `rank` is accepted. A keyset cursor at
  `rank = 1` cannot distinguish remaining rows with the same rank, so a
  conforming lexicographic adapter may skip survivors.
- **Test gap:** `src/data-directory/server/directory-page.test.ts:96-129`
  uses four rows with the minimum page size of 25, manually injects only a
  forward cursor, and never obtains or follows an emitted next/previous cursor.
  Its concurrent-delete case performs a first-page read with no cursor, so it
  cannot prove no skip/duplicate across requests.
- **Impact:** the generic foundation accepts precisely the unstable ordering
  that the original request and fixed plan require it to prevent.
- **Required repair:** make the terminal immutable unique field an explicit,
  validated order contract, reject an order without it before adapter I/O, and
  add more-than-25-row forward/backward traversal tests through emitted cursors,
  including duplicate sort values and boundary insert/delete behavior.

### B2 — fresh evidence is not verifier-complete

- **Evidence:** fresh production Chromium run `20260724041925` passed both
  Playwright tests and generated all 24 six-theme captures at
  320/375/768/1440. Direct inspection found no visual severity 2+ issue, and
  its manifest SHA-256 is
  `fec769c87ad79e872dea39b4cc39cab73b8d30a6fbca9b602c8eb458b32d6ba0`.
  `node scripts/verify-design-system-evidence.mjs` nevertheless exits nonzero:
  `artifacts/design-system/20260724041925/review.md` is absent.
- **Impact:** AC5's final binding cannot distinguish inspected current captures
  from an unreviewed run.
- **Required repair:** write the current run's review bound to the exact run ID
  and manifest hash, record the visual/a11y loop and limits, then rerun the
  evidence verifier successfully. Regenerate only if the UI changes.
