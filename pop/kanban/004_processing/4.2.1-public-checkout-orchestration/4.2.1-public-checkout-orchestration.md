# 4.2.1-public-checkout-orchestration — Public checkout orchestration

- **Phase:** [[roadmap/4-checkout-and-orders|Phase 4.2]]
- **Plan:** [[4.2.1-public-checkout-orchestration.plan]] · **Approval:** [[4.2.1-public-checkout-orchestration.approval]] · **Verification:** [[4.2.1-public-checkout-orchestration.verify]]

## What

Create secure server-side public checkout orchestration that validates an active link, stores its required customer snapshot, creates one Nautt payment request, and returns only the redacted payment capability.

## Why

The public UI and owner order views need a trusted order/payment creation boundary with no browser-supplied owner, price, provider UUID, or credential.
