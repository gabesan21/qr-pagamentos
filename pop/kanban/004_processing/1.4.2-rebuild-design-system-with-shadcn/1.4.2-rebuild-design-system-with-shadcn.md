# 1.4.2-rebuild-design-system-with-shadcn — Rebuild the design system with shadcn

## What

Audit and rebuild the existing visual foundation as a professional, modular shadcn-based system. Establish pinned configuration and dependencies, self-hosted/offline-safe typography, coherent semantic token scales, intentional component APIs and states, responsive composition rules, an updated `DESIGN.md`, and an authoritative `/design-system` specimen without anticipating later product surfaces.

## Why

The current primitives and system-font presentation are too basic and inconsistently consumed; later Phase 1.4 screens need one dependable component contract before they are recomposed.
