# Group g04 - Browser evidence and closeout - [[1.4.2-rebuild-design-system-with-shadcn]]

- **Order:** 4.
- **Depends on:** g03.
- **Description:** Prove responsive, accessible, offline-safe rendering and synchronize design memory/contracts.

## Movements

- [ ] (agent) Add a deterministic Playwright webServer project and `pnpm design-system:evidence` runner for `/design-system` with external-request rejection, console/page-error capture, `document.fonts.ready`, focus traversal, no-overflow checks, and axe scanning.
  - Expected: the runner creates a new run ID/start time, invalidates the old review binding, and each theme/viewport case is isolated and exits nonzero on any required failure.
  - If it fails: route unexpectedly reaches the database -> verify cookie-free session short-circuit and fix only specimen setup; do not weaken auth services or mock the browser page.
- [ ] (agent) In each of the eight cases, first assert marker coverage for every visible enabled button/input/select/status/prose contract; then assert all `data-ds-hit-target` boxes are >=44 by 44 CSS pixels, computed body font contains IBM Plex Sans, each `data-ds-section` has <=1 `data-variant="default"` button, each `data-ds-status` has a visible `data-ds-status-cue`, and each `data-ds-prose` width is <= a browser-created 65ch reference.
  - Expected: machine-readable results record the measured values/selectors for every case before the corresponding screenshot is accepted.
  - If it fails: missing marker or failed measure -> fix the component/specimen contract and rerun the entire fresh evidence command; never waive one viewport/theme.
- [ ] (agent) Capture light/dark PNGs at 320, 375, 768, and 1440, then write a manifest containing run ID, start time, git HEAD, assertion-results hash, and each PNG path/size/SHA-256.
  - Expected: exactly eight newly created non-empty PNGs have requested widths, mtimes after run start, deterministic names, matching hashes, and no stale file survives the run.
  - If it fails: font/layout readiness or nondeterminism -> await fonts/network idle, disable decorative motion via reduced-motion, clear that run, and regenerate all eight rather than reuse partial captures.
- [ ] (agent) Perform screenshot -> UI review against the exact manifest hashes, correct and recapture all eight up to two loops, then write `review.md` with the final run ID and manifest SHA-256.
  - Expected: `review.md` records inspected hashes, findings, fixes, and final independent severity with no unresolved severity 2+ issue; `pnpm design-system:evidence:verify` validates the binding.
  - If it fails: a severity 2+ issue survives two loops -> mark blocked with screenshots and exact selector/viewport; do not declare subjective acceptance.
- [ ] (agent) Update `DESIGN.md`, root/new/legacy DOX contracts, the design-system spec Open section, and planned administrative-foundation closeout notes to match exact sources, adapters, inventory, and evidence.
  - Expected: no stale system-font/no-browser/custom-only primitive claim remains; DOX indexes and reference triggers resolve; `pop_validate` is clean.
  - If it fails: source and docs disagree -> source is not silently treated as truth; reconcile within approved scope or record divergence and return to 002.
- [ ] (agent) Run focused tests, `pnpm design-system:evidence`, review/bind that fresh run, `pnpm design-system:evidence:verify`, `pnpm check`, then inspect the final diff for generated/untracked artifacts and secrets.
  - Expected: all acceptance runs pass independently, the build is offline-font-safe, only intended source/evidence/docs are tracked, and the worktree is ready for 005.
  - If it fails: classify local defect vs environment failure, apply only an approved fork/countermeasure, and re-run the complete affected gate.
