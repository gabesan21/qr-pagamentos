## Movements

- [x] (agent) Review `container/*.mjs`, `Dockerfile`, and `compose.yaml` for explicit lifecycle/error flow, direct-child signal behavior, secret redaction, database preflight, pinned images, non-root execution, and startup ordering; update only this group's ledger below.
  - Expected: every container surface has evidence/disposition and all application-contract invariants remain explicit.

- [x] (agent) Review `install/install.sh`, `install/uninstall.sh`, `install/test.sh`, `.env` examples, and relevant `README.md` operator instructions for duplicated parsing/rules, clear failure paths, dry-run fidelity, secret handling, default data preservation, and absence of privilege escalation behavior.
  - Expected: each installer/documentation surface has evidence/disposition and operator-visible behavior remains consistent.
  - If it fails: remediation changes prerequisites, destructive defaults, or needs operator judgment → scope/safety change → abort; otherwise extend `install/test.sh` first.
- [x] (agent) Review `pop/scripts/container-{contract-check,prepare-secrets,test}.mjs` as the executable deployment contracts, especially scenario isolation and cleanup; classify findings in this group's ledger.
  - Expected: harness intent and failure diagnostics are locally understandable, and cleanup targets only uniquely labeled disposable resources.
  - If it fails: cleanup identity cannot be proven → destructive-test risk → do not run the scenario and abort.
- [x] (agent) Apply each confirmed local remediation as one cohesive transformation with a protecting test; after each, run `pnpm container:contract-check` and the smallest relevant installer/container check.
  - Expected: checks exit 0 and each diff maps to one ledger finding without secret-bearing or floating configuration.
  - If it fails: contract/scenario regression → correct the same transformation from captured evidence; a broad deployment redesign becomes tracked debt and is not implemented.
- [x] (agent) Finalize all deployment/installer ledger rows, including context, impact, and next step for any tracked debt.
  - Expected: no unresolved candidate remains and no operator behavior changed silently.
  - If it fails: candidate cannot be classified or documentation disagrees with executable behavior → sync the local documentation only if behavior is already approved; otherwise abort.
