# Plano — [[0.1.4-fix-pay-locale-test-copy]]

- **Etapa:** 002_planning · **Responsável:** agent planejador

## Objetivo e resultado esperado

- **Objetivo:** corrigir a asserção de locale no teste "renders a generic unavailable state without a form" de `src/app/pay/[identifier]/page.test.tsx`, que espera copy em inglês ("This payment link is unavailable") enquanto o visitante sem sessão resolve para o locale padrão `pt-BR` (`defaultLocale` em `src/i18n/locales.ts`).
- **Resultado observável:** o arquivo de teste focado fica verde e `pnpm check` volta a passar integralmente em develop.

## Estratégia

A decisão raiz é: **o teste está errado, o código está certo.** O contrato i18n (root `AGENTS.md`, "Application contract") manda rotas públicas não localizadas resolverem a preferência persistida `pt-BR`/`en`, com `pt-BR` como padrão quando nada está persistido. O próprio primeiro teste do mesmo arquivo já prova essa intenção: no cenário não autenticado ele exige `read(identifier, "pt-BR")`. O segundo teste é o único ponto divergente — foi escrito com copy em inglês e nunca atualizado quando o default passou a ser pt-BR. A correção é pontual: alinhar a asserção do estado indisponível ao dicionário pt-BR (`checkoutUnavailableHeading`: "Este link de pagamento está indisponível"). Nenhuma mudança de comportamento, página, dicionário ou spec — somente o teste.

## Áreas afetadas
