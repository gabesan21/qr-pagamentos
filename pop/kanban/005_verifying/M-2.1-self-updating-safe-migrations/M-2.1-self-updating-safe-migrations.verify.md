## Findings

### Accepted concern — baseline co-mutation regression does not test the prior bypass

- **Location:** `pop/scripts/migration-policy-contract.mjs:128-136`.
- **Evidence:** the test changes baseline SQL to `-- co-mutated baseline SQL\n` but writes its inventory entry as `"b".repeat(64)`, not the SHA-256 of those changed bytes. The old self-authorizing verifier returned only when the changed SQL and editable inventory hash matched; it would reject this fixture too because the two values already disagree. The new immutable `BASELINE_INVENTORY_SHA256` anchor in `migration-policy.mjs:10-13` is a sound implementation-level correction, but this test cannot regress that exact exploit.
- **Impact:** a future refactor could remove/bypass the independent anchor while retaining this test green.
- **Disposition:** explicitly accepted by the human on 2026-07-22; no further change is authorized for this task.
