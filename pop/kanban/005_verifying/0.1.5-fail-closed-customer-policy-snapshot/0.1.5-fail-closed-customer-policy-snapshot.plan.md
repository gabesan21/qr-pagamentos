# Plano — [[0.1.5-fail-closed-customer-policy-snapshot]]

- **Etapa:** 002_planning · **Responsável:** agent planejador

## Objetivo e resultado esperado

- **Objetivo:** Make the default branch of `toPolicySnapshot` (`src/orders/order-view.ts`) fail-closed: a persisted `checkoutDataPolicy` value outside the closed `CHECKOUT_DATA_POLICIES` enum must yield the all-null `CustomerSnapshotV1` (no name, email, cpf, or address) instead of the current full `NAME_EMAIL_CPF_ADDRESS` exposure. Only `NAME_EMAIL_CPF_ADDRESS` keeps the full tuple; every other value — known or unknown — that is not explicitly matched exposes nothing beyond its own tuple.
- **Resultado observável:** A new regression test in `src/orders/order-view.test.ts` proves an unknown/corrupted policy value exposes zero customer data; the full `src/orders` suite and `pnpm check` stay green.

## Estratégia

Single-point change: `toPolicySnapshot` already enumerates four of the five enum members explicitly; the fall-through `return` currently assumes the fifth. Make the fifth member (`NAME_EMAIL_CPF_ADDRESS`) an explicit branch and turn the fall-through into the blank snapshot, so any value outside the closed set — corrupted row, drifted schema, future enum member added without updating this projection — degrades to no exposure rather than maximum exposure. No signature, type, or caller change: `CheckoutDataPolicy` is a closed union, so an unknown value only reaches this code via a cast or a persisted string; the regression test must therefore exercise the default through an explicit cast, mirroring how corrupted data would arrive.

Spec decision: `pop/specs/checkout-and-order-lifecycle.md` (Task 4.3.1 slice) states the projection exposes "the exact policy-shaped `CustomerSnapshotV1`" but is silent on out-of-enum persisted values — silence on a customer-data boundary is ambiguity. Add exactly one clarifying line to that slice recording the fail-closed invariant; no other spec or contract text changes. Root `AGENTS.md` already describes the projection as policy-exact and needs no edit.

## Áreas afetadas

- `pop/specs/checkout-and-order-lifecycle.md` — one-line clarification of the fail-closed invariant in the Task 4.3.1 slice.
