# Group g02 — Authenticate and parse — [[2.3.1-verify-and-handle-nautt-webhooks]]

- **Order:** 2
- **Depends on:** g01
- **Description:** Verify bounded exact bytes against owner secrets before parsing and normalize authenticated envelopes.

## Movements

- [x] (agent) Add a server-only verification candidate reader selecting only `ACTIVE` rows with non-null encrypted secrets; decrypt each candidate, HMAC the unchanged `Buffer`, compare fixed 32-byte digests with Node `timingSafeEqual` without early return, clear plaintext references, and accept exactly one owner.
  - Expected: all candidates are checked in stable behavior; zero/multiple matches return one opaque authentication result and perform no JSON parse/write/API-key decrypt/provider fetch.
  - If it fails: candidate selection requires parsing or undocumented routing → trigger the plan abort; do not trust the envelope to choose a secret.
- [x] (agent) Add a single-consumption bounded body helper: cheap optional `Content-Length` precheck that cancels an untouched oversized body, then one `request.body.getReader()` loop that counts before copying, retains one byte-for-byte copy of each bounded chunk, cancels/releases/discards immediately above 256 KiB before concatenation, and returns one byte-exact bounded `Buffer`; never use `arrayBuffer()` or decode during this phase.
  - Expected: accepted multi-chunk/no-length bytes are identical at HMAC; declared or chunked oversize returns empty `413`; a no-`Content-Length` stream cancellation spy fires on the first crossing chunk and unread tail chunks are not pulled; candidate decrypt/HMAC, parser, delivery store, API-key decrypt, and fetch all remain at zero.
  - If it fails: framework request bodies cannot expose/cancel the reader → stop and return to 002; do not fall back to full buffering or treat `Content-Length` as enforcement.
- [x] (agent) Add strict signature/header/envelope helpers: one `sha256=[0-9a-f]{64}` value, UUID delivery, closed event set, and post-auth envelope/header equality with normalized timestamp/order UUID/attempt evidence; do not require undocumented transport headers.
  - Expected: whitespace/reordering/mutation, comma-joined duplicate signatures, bad length/case/hex, mismatch, and invalid JSON fail in their documented phase after only the allowed prerequisites.
  - If it fails: framework header normalization obscures duplicates → reject any combined value outside the single-value grammar and prove it in route tests; do not accept a list.
- [x] (agent) Add streaming/signature/intake unit tests with controlled chunked streams, cancellation and dependency spies, injected crypto/parser/store/fetch clocks, plus the 1,000-candidate capacity/cost case.
  - Expected: exact fixture, byte-preserving chunks, early oversize cancellation/no downstream work, comparison count, zero-side-effect auth failures, redaction, and `<14.5s` active path with injected 10s GET all pass deterministically.
  - If it fails: real-clock benchmark is flaky → use injected monotonic operation costs for the gate and record a separate non-gating local measurement; never weaken semantic counts or the 15-second bound.

## Execution notes

- Record candidate count only in test evidence; production errors/logs must not reveal owner IDs, secrets, signatures, or active-registration counts.
- Evidence: exact fixture/mutation, malformed grammar, all-candidate/ambiguous matching, byte-preserving chunks, first-crossing cancellation, zero-side-effect auth failures, and the 1,000-candidate + injected 10-second cost gate pass without production logging.
