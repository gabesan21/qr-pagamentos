## Blocking findings

### B1 — The declared DTCG resolver does not resolve the runtime token contract

`src/design-system/tokens/resolver.json:3` declares `"version": "2026.07"` instead of the required DTCG Resolver version `"2025.10"`. Lines 4–20 load the entire `motion` group as a shared set but define no motion modifier/default/context, so `full` and `reduced` coexist rather than resolving to one value.

The runtime projection never consumes that resolver: `scripts/generate-design-tokens.mjs:10-55` reads only `themes.tokens.json`, uses a hardcoded semantic map, and emits only theme colors. The source motion values at `themes.tokens.json:138-145` therefore do not own the handwritten runtime values at `globals.css:61-62` (or the reduced-motion override). `scripts/design-tokens.test.ts:25-58` neither parses the resolver nor checks version, references, cycles, or motion contexts.

**Required correction:** make the resolver DTCG-valid, model deterministic full/reduced motion selection, and have generation/tests consume and verify the resolved semantic graph (including unresolved references and cycles), rather than maintaining a parallel mapping.

### B2 — “In gamut,” fallback, and canonical-source guarantees are not enforced

`scripts/design-tokens.test.ts:35-49` checks only OKLCH component bounds and hex syntax, then computes contrast exclusively from hex. It never converts OKLCH to sRGB, checks gamut, or proves that the fallback encodes the same color. Independent conversion identified seven out-of-sRGB source colors:

- `pix-paper`: `primary`, `primary-hover`
- `cashier-daylight`: `primary`, `primary-hover`, `success-foreground`
- `settlement-sand`: `card`
- `vault-blue`: `success`

The authored fallbacks also materially diverge from their OKLCH values—for example, `pix-paper.primary` declares `#006b5b` while the clipped OKLCH conversion is approximately `#006145`. The test consequently proves contrast for a different palette from the one modern browsers render.

Separately, `globals.css:8-65` retains a handwritten reference-token source, while `scripts/check-design-tokens.mjs:21-25` explicitly removes both that block and the generated block before linting. Raw values and drift can therefore persist inside a purported canonical source without a gate failure.

**Required correction:** use gamut-safe OKLCH values, derive/validate equivalent sRGB fallbacks and contrast from the rendered values, and project or strictly verify all reference/semantic/type/motion tokens from one canonical graph.
