## Fixed contract

### Server, scope, and URL

- `src/data-directory/` exposes only merchant and administrator query entry points. Each receives the resolved active principal, rechecks the exact role before adapter I/O, and supplies either the server-derived owner ID or explicit global-read scope. No unscoped runner, cross-role fallback, URL/cursor scope, shared business projection, or cursor authority exists.
- Each future domain registers a closed directory ID, filter schema, redacted row DTO, and deterministic keyset order of immutable fields ending in a unique stable ID. The adapter combines scope, filters, and the exact lexicographic seek predicate, requests `pageSize + 1`, and returns at most `pageSize` rows plus next/previous cursors. Surviving rows do not duplicate/skip while traversing one direction; cross-request snapshot isolation is not promised, so concurrent inserts/deletes may appear/disappear naturally. Offset, total count, arbitrary sort/column input, raw query fragments, and client-side full-list filtering are excluded.
- Page sizes are exactly `25`, `50`, `100`; default `25`. Before constructing `URL`/`URLSearchParams`, percent-decoding, `+` conversion, or Unicode work, measure the exact query substring from the request target (bytes after `?`, excluding `?`/fragment) as UTF-8 with `Buffer.byteLength`: â‰¤2048 bytes is eligible and â‰Ą2049 is invalid. Then require well-formed `%HH` escapes and fatal UTF-8 decoding; replacement decoding is forbidden. After decode, accept at most 32 entries, scalar uniqueness, and at most eight registered filters of: (a) Unicode-trimmed nonblank text, no controls/line breaks, â‰¤100 code points, never truncated; or (b) exact closed ASCII enum values, â‰¤5 distinct values, de-duplicated/sorted. The separate cursor payload bound remains â‰¤512 decoded bytes.
- URL keys are only `q`, registered `filter.<name>`, `pageSize`, and `cursor`. Valid input is serialized deterministically as `application/x-www-form-urlencoded` in fixed key/value order, omitting empty filters/default size. Compare this canonical ASCII query byte-for-byte with the raw query; inequality returns a relative same-path `307` before adapter I/O, and equality proceeds without redirect, making the redirect idempotent/loop-free. The native GET filter/size form never submits a cursor. An authentic old cursor retained with changed canonical filters/size is removed by a `307` to page one; pagination links retain current filters/size. Invalid/over-limit input performs no adapter I/O, echoes nothing, and renders the generic reset state instead of redirecting from attacker-controlled data.
- A cursor is a canonical-payload-plus-HMAC base64url token whose entire decoded payload-and-tag representation is â‰¤512 bytes. Its canonical envelope has exact keys: version, directory, closed non-identifying `scopePurpose` (`MERCHANT_OWN | ADMIN_GLOBAL`), direction, size, canonical-filter SHA-256, order ID, and validated ordering tuple. HMAC-SHA-256 covers every payload byte and is compared timing-safely before tuple use. The codec derives a purpose/scope-specific MAC key with HKDF-SHA-256 from the already-required 32-byte server-only `NAUTT_ENCRYPTION_KEY`: domain label `qr-data-directory-cursor-v1`, purpose, and the resolved principal ID for `MERCHANT_OWN`, or the constant `admin-global` for `ADMIN_GLOBAL`. The ID participates only in server-side key derivation and never appears in the token. This domain-separated use performs no Nautt decrypt/provider work and adds no secret/config/deployment field. Wrong purpose, another merchant, changed payload/tag, or another key returns the same invalid-query state. Every request still re-authorizes and reapplies server scope; cursors grant no authority.

## Risks and aborts

- Authorization bypass: role-specific public entry points, defense-in-depth checks, server-derived owner scope, and zero-call wrong-role tests.
- Duplicate sort values: immutable lexicographic tuple ending in unique ID; forward/reverse boundary tests across inserts/deletes.
- Cursor/filter injection or cross-scope replay: pre-decode raw-byte bounds, strict decoding, registered fields/orders, authenticated purpose/scope binding, parameterized adapter values, and no cursor authority.
- Responsive semantic loss/duplicate controls: native wide table, narrow labelled facts, one active accessibility tree, keyboard/axe/screenshots.
- Registry drift: pinned CLI info/docs/dry-run/diff, source review, token lint, inventory update, no overwrite.
- Abort if official Empty/Pagination source cannot be established and existing inventory cannot satisfy the contract without duplication; record the exact CLI limitation.
- Return to 002/owner task if a consumer requires offset/count, arbitrary sorting, a new filter kind, or concrete projection.
