# Plan — [[6.3.1-build-pagination-filter-table-foundation]]

- **Stage:** 002_planning · **Responsible:** planning agent

## Objective and result

- Establish one bounded server-directory contract and one accessible reusable data-directory composition for future administrator and merchant directories.
- A bilingual synthetic specimen proves merchant-scoped and administrator-global use through the same role-neutral UI, stable keyset windows, bounded filters, canonical URLs, and all responsive states in six themes.
- Do not create or migrate an Orders, Links, Users, Products, Categories, analytics, or other concrete business directory.

## Fixed contract

### Server, scope, and URL

- `src/data-directory/` exposes only merchant and administrator query entry points. Each receives the resolved active principal, rechecks the exact role before adapter I/O, and supplies either the server-derived owner ID or explicit global-read scope. No unscoped runner, cross-role fallback, URL/cursor scope, shared business projection, or cursor authority exists.
- Each future domain registers a closed directory ID, filter schema, redacted row DTO, and deterministic keyset order of immutable fields ending in a unique stable ID. The adapter combines scope, filters, and the exact lexicographic seek predicate, requests `pageSize + 1`, and returns at most `pageSize` rows plus next/previous cursors. Surviving rows do not duplicate/skip while traversing one direction; cross-request snapshot isolation is not promised, so concurrent inserts/deletes may appear/disappear naturally. Offset, total count, arbitrary sort/column input, raw query fragments, and client-side full-list filtering are excluded.
- Page sizes are exactly `25`, `50`, `100`; default `25`. Accept at most 32 entries and 2 KiB of directory query data. Scalars occur once. At most eight registered filters use only: (a) Unicode-trimmed nonblank text, no controls/line breaks, ≤100 code points, never truncated; or (b) exact closed ASCII enum values, ≤5 distinct values, de-duplicated and sorted.
- URL keys are only `q`, registered `filter.<name>`, `pageSize`, and `cursor`. Unknown, malformed, duplicate scalar, or over-limit input yields one typed invalid-query result before adapter I/O. Empty filters/default size are omitted. Native GET forms make the URL authoritative; filter/size changes drop the cursor, while pagination retains canonical filters/size.
- A cursor is a versioned base64url envelope, decoded ≤512 bytes, with exact keys: directory, direction, size, canonical-filter SHA-256, order ID, and validated ordering tuple. It contains no principal, owner, role, secret, row DTO, path, or arbitrary filter data. Mismatch or malformed input gets the same generic reset state. Every request re-authorizes and reapplies server scope.

### UI, i18n, responsiveness, and states

- Reuse owned `Table`, `FieldGroup`/`Field`, `Input`, `NativeSelect`, `Button`, `Alert`, `Skeleton`, `Spinner`, `Card`, and `Separator`. Before registry-source edits, run pinned shadcn `info`, `docs`, and `add --dry-run`/`--diff`; add only official `Empty` and `Pagination` sources if their Radix/nova APIs are confirmed. Never hand-roll/fetch substitutes, overwrite local sources, create visual variants, or branch by role/theme.
- The composition receives localized copy, redacted rows, column/fact descriptions, canonical URLs, and optional consumer actions; it imports no business store/auth service. The labelled GET toolbar provides search, registered enum controls, page size, submit, and reset with one primary action. Pagination is a labelled landmark of real links; unavailable actions are absent or natively disabled.
- At 320/375, rows are ruled `dl`/`dt`/`dd` facts; at 768/1440, one captioned native table with scoped headers. One definition feeds both, CSS leaves only one renderer/action set in the accessibility tree, DOM/focus/reading order match, text wraps, numeric facts use tabular figures, targets are ≥44×44px, and the page never overflows (wide-table overflow stays in its labelled keyboard-reachable container).
- Closed mutually exclusive states: `ready`; `loading` (`aria-busy`, polite localized status, shape-stable skeletons); `empty` (optional single CTA); `filtered-empty` (clear filters); `invalid-query` (generic reset, no echoed input); `error` (generic retry, no exception/scope detail). Existing primitive focus/disabled/reduced-motion rules apply.
- Generic copy is one new `data-directory` domain with exact `en`/`pt-BR` parity and no concrete-domain copy.
- Parse/auth/scope finish before I/O. Success makes one bounded data query and no count; invalid/wrong-role makes zero calls. RSC serializes only the current redacted window and URLs; add no required client fetch/cache, shared mutable state, or browser storage.

## DAG and topology

1. F01 fixes server result, cursor, URL, and scope types.
2. F02 consumes only those public types for generic UI/specimen.
3. F03 synchronizes contracts/evidence and runs focused/aggregate gates.

Use one execution orchestrator, three serialized write sets, and one front executor at a time. Keep the visual/evidence lane separate from other UI tasks; this task has no Prisma, media, Compose, or installer lane. Use medium executors, strong critical 003, and strong full 005.

## Risks and aborts

- Authorization bypass: role-specific public entry points, defense-in-depth checks, server-derived owner scope, and zero-call wrong-role tests.
- Duplicate sort values: immutable lexicographic tuple ending in unique ID; forward/reverse boundary tests across inserts/deletes.
- Cursor/filter injection: exact bounded decoder and registered fields/orders with parameterized adapter values; cursors grant no authority.
- Responsive semantic loss/duplicate controls: native wide table, narrow labelled facts, one active accessibility tree, keyboard/axe/screenshots.
- Registry drift: pinned CLI info/docs/dry-run/diff, source review, token lint, inventory update, no overwrite.
- Abort if official Empty/Pagination source cannot be established and existing inventory cannot satisfy the contract without duplication; record the exact CLI limitation.
- Return to 002/owner task if a consumer requires offset/count, arbitrary sorting, a new filter kind, or concrete projection.
