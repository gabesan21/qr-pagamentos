# Plan — [[5.2.1-public-storefront-page]]

- **Stage:** 002_planning · **Owner:** planning agent

## Objective and expected outcome

- **Objective:** Publish the opt-in owner storefront at `/store/[slug]`, using the persisted 5.1.1 settings, and list only redacted active products that retain an active, unexpired payment link.
- **Observable outcome:** A visitor receives the configured localized store name and accent plus localized product title, description, price, and a link to `/pay/[identifier]`; disabled, unknown, malformed, or non-public stores resolve to one opaque unavailable view.

## Strategy

Implement one server-only public projection, separate from owner services and checkout presentation. It validates the slug before storage, requires the owner storefront to be enabled, and selects only the public settings plus qualifying product/payment-link facts. It must preserve the approved 5.1.1 contract exactly: global `/store/` slug namespace, `^[a-z0-9](-?[a-z0-9])*$` format and its accepted bounds, nullable bilingual display names with public fallback here, validated `#RRGGBB` accent, and disabled-by-default/enable-requires-slug semantics. It must not repair, extend, or expose owner settings.

The route is sessionless and uses the existing persisted locale only when a valid cookie principal exists; otherwise it uses `pt-BR`. It has no mutation/API surface. The store page reuses the receipt-ledger primitives and scoped semantic CSS. The persisted accent may supply only the scoped `--storefront-accent` semantic value after the projection has validated it; no raw UI literal or general inline visual-style exception is introduced. Both dictionaries receive parity-tested public-store copy, and `DESIGN.md` records the storefront state matrix and accent boundary.

## Affected areas

- `src/storefront/public-storefront.ts` (+ tests) — redacted, server-only public projection and Prisma read.
- `src/app/store/[slug]/page.tsx` (+ tests) — public route, locale resolution, unavailable and product-list presentation.
- `src/app/globals.css`, `DESIGN.md` — scoped storefront composition using existing semantic tokens.
- `src/i18n/dictionaries/{en,pt-BR}.ts`, `src/i18n/dictionaries.test.ts` — bilingual copy and parity coverage.

## Risks and abort conditions

- **Risk:** a public query accidentally broadens the checkout or owner data boundary — mitigate with a closed projection type, explicit Prisma `select`, and negative redaction assertions.
- **Risk:** dynamic branding bypasses the token lint or leaks unvalidated input into CSS — mitigate by accepting only the 5.1.1 validated accent through a single scoped semantic custom property with platform fallback.
- **Abort if:** 5.1.1 is not integrated, its spec is missing, or required public facts need a schema/migration/owner-settings change; mark the task blocked with the missing contract rather than extending the dependency.
