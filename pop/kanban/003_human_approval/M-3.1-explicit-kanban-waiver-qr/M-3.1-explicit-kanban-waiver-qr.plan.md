# Plan — [[M-3.1-explicit-kanban-waiver-qr]]

- **Stage:** 002_planning · **Owner:** agent planner

## Objective and expected result

- **Objective:** make the standalone qr-pagamentos harness reject implicit kanban waivers while preserving narrowly scoped, unequivocal human overrides.
- **Observable result:** ordinary execution language never bypasses the workflow; a request without a card materializes and advances a task; yolo follows its complete route; an explicit kanban bypass still leaves an auditable ledger.

## Strategy

Align the local instructions, workflow, orchestration skills, and templates around one precedence rule: a human command overrides only the rule or gate it explicitly names. Define common phrases such as “apply”, “execute”, “urgent”, “until finished”, or “in yolo” as execution intent, not waiver language; only an unequivocal instruction such as “without kanban” activates the deviation protocol.

Make missing-card requests enter `new-task` and then `advance-task`. Clarify that “start the yolo flow” materializes, releases, and traverses the same kanban, with the existing non-critical 002→004 route treated as a legitimate transition rather than a skipped stage.

An explicit bypass may waive the stages, but not continuity: it must retain an identifiable memory/ledger with the authorizing command, result and commit, plus a recorded specs/DOX impact assessment; those contracts are edited only when affected. Preserve qr-pagamentos-specific rules and English project content while leaving vendored shared workflow and template prose in pt-BR.
