## Round 1 — 2026-07-24

### Resposta do crítico (yolo)

- **Context:** strong independent.
- **Return:** 1 of 2.
- **Decision:** returned → 002.
- **Original request:** the plan remains within the requested shared pagination/filter/table foundation, preserves role separation, and correctly excludes concrete Orders, Links, Users, and other business directories.
- **Resolved strengths:** deterministic immutable keysets end in a unique ID; forward/reverse and mutation fixtures are executable; query/filter cardinalities, cursor decoding, role-denial zero-I/O, RSC redaction, six closed states, one responsive accessibility tree, official shadcn preflight, bilingual parity, six-theme evidence, serialized front ownership, and critical full verification are all proportionate and observable.
- **Concrete blockers:**
  1. Cursor context binding excludes the authorization context it must reject. The exact envelope carries directory/filter/size/order data but explicitly carries no role, owner, or other scope discriminator, while one domain directory is intended to serve both merchant-own and administrator-global entry points. Consequently a valid global cursor can be replayed into the merchant entry point (or vice versa) and be accepted as a seek position rather than the promised generic reset. Re-authorization prevents disclosure, but does not satisfy cursor context binding or opacity. Define a non-identifying, server-validated scope-purpose binding (for example separate closed role-scope directory purposes and an integrity/authenticity mechanism if the cursor is called opaque) and add cross-role/scope replay plus tamper tests to AC1/AC2; do not encode an owner or principal identifier.
  2. The canonical-URL and native-GET-form contracts are not jointly executable as written. A named page-size select at its default submits `pageSize=25`, and named empty text controls submit empty values, yet the fixed contract requires default size and empty filters to be absent. Parsing to a canonical value does not canonicalize the browser URL. Specify the observable canonicalization mechanism (such as an opaque same-route redirect before directory I/O, with loop-free canonical comparison), its no-echo/error behavior, and tests proving native filter/size submissions both drop cursors and leave the address bar at the canonical URL.
  3. The global 2 KiB query limit is not defined in measurable terms. The parser/AC currently cannot decide whether this means encoded URL bytes, decoded UTF-8 bytes, UTF-16 code units, or canonicalized output, especially for percent-encoded and non-ASCII inputs; an implementation could pass its own tests while accepting materially different bounds. Define the exact byte representation and when the limit is checked, then add boundary cases for percent expansion and multibyte Unicode. Keep the separate 512 decoded-byte cursor bound.
