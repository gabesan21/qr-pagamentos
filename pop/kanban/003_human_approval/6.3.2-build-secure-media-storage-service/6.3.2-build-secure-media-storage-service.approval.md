### Yolo critic response

- **Context:** strong independent.
- **Return:** 1 of 2.
- **Decision:** returned → 002.
- **Concrete reasons:**
  1. The decoder is still an executor choice: neither package/version nor the enforceable configuration is named, and no preflight proves single-frame detection plus dimension/pixel/resource rejection before unsafe allocation in the pinned Node/container environment. This is the plan's own abort condition, so it must be resolved before execution and made observable in AC3.
  2. The declared metadata has no monotonic revision/fencing field, while the strategy requires conditional state/version transitions and AC5 says stale activate/orphan/delete calls cannot win. A state-only compare permits ABA after an object is orphaned and reactivated; the plan must define the token carried by every lifecycle claim and test that exact race.
  3. `ACTIVE` responses are immutable-cacheable, yet the same identifier can later become `ORPHANED` and be purged after 24 hours. Previously cached bytes then remain publicly available while the contract says only `ACTIVE` media is public. Choose a permanent-public immutable identifier contract or a revocable cache policy, synchronize the lifecycle/spec, and add a transition assertion to AC7.
  4. The quota is described as retained-object/retained-byte protection but excludes overdue `ORPHANED` and `DELETING` files while reconciliation is only a bounded callable service with no runner in this task. Delayed cleanup can therefore exceed the physical 100 MiB/256-object boundary and admit more media. Define whether all still-present bytes count or make successful reclaim a fail-closed admission prerequisite, then cover delayed-cleanup and concurrent-admission races.
  5. AC4 proves outcomes but not the promised crash-durable, atomic no-overwrite protocol, and AC7 does not require an opened-descriptor `O_NOFOLLOW`/`fstat` read that closes the check/open symlink race. Name the supported filesystem semantics and add injectable crash/call-order plus link-swap assertions for file sync, directory sync, no-clobber publication, and descriptor-bound streaming.
