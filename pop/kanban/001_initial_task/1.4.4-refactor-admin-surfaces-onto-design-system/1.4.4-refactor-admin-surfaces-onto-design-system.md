# 1.4.4-refactor-admin-surfaces-onto-design-system — Refactor admin surfaces onto the design system

- **Phase:** [[roadmap/1-administrative-foundation|Phase 1.4]]
- **Plan:** [[1.4.4-refactor-admin-surfaces-onto-design-system.plan]] · **Approval:** [[1.4.4-refactor-admin-surfaces-onto-design-system.approval]] · **Verification:** [[1.4.4-refactor-admin-surfaces-onto-design-system.verify]]

## What

Decompose the existing administration page into cohesive reusable compositions and migrate account management, payment settings, language preference, notices, navigation, and logout onto the approved shadcn layer. Remove duplicate raw controls and page-local visual variants while preserving server authorization, mutations, and `pt-BR`/`en` behavior.

## Why

The current administrative page concentrates unrelated responsibilities and inconsistently bypasses the existing primitives, making the interface difficult to refine and future tasks likely to drift.
