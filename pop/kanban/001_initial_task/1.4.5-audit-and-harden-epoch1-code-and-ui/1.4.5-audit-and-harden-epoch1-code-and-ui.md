- No undocumented waiver or no-capture fallback completes this task. If disposable infrastructure or browser capability is unavailable, block with the exact missing capability and recovery command.
- Seed findings from 1.4.3 verification (2026-07-17): (a) `/login` pending submit state uses plan-mandated `useFormStatus` that cannot fire with the native string form action — repair needs a client-driven pending state preserving the POST contract; (b) the ESLint config does not ignore `pop/worktrees/**`, so `pnpm check` in the main clone fails while any task worktree exists — both confirmed severity-2 advisories to remediate here.
