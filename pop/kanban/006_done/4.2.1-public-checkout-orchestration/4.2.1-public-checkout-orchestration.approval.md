# Approval — [[4.2.1-public-checkout-orchestration]]

## Decision brief

- **Delivery:** sessionless, no-store public checkout orchestration with policy-exact private snapshots, link-scoped retry idempotency, one trusted Nautt quote/onramp attempt, durable local/provider attachment, indeterminate preservation, and a redacted QR/status capability.
- **Trust and privacy:** active-link state derives owner, product, exact money/pair, policy, and credential server-side; the request supplies only a bounded retry key and exact snapshot.  PII, provider/local identifiers, raw errors, secrets, and cacheable responses never cross the boundary.
- **Atomicity/recovery:** one link/key maps to one attempt and at most one POST; `SINGLE_USE` remains settlement-consumed only.  A post-dispatch failure is `INDETERMINATE`; only a known durable provider UUID can reach existing authoritative one-read recovery, never a recreate.
- **Exclusions:** no checkout/status UI, dictionaries, browser flow, public status/poll route, protected order view, webhook/poll/recovery execution change, provider retry, or undocumented provider behavior.
- **Topology/proof:** F01 durable orchestration then F02 HTTP mapping; run focused deterministic tests, `pnpm db:generate`, `pnpm db:contract-check`, `pnpm check`, and diff check.  Docker, `db:test`, and browser/runtime are authorized deferred evidence.
- **Specs:** [[specs/checkout-and-order-lifecycle|Checkout and order lifecycle]] is draft pending this independent 003 approval; Nautt and catalog specs remain authoritative for their existing concerns.
- **Plan:** [[4.2.1-public-checkout-orchestration.plan]] — follow for ownership, abort conditions, and acceptance criteria.
