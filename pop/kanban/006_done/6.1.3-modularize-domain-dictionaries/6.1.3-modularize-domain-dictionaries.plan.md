# Plan — [[6.1.3-modularize-domain-dictionaries]]

- **Stage:** 002_planning · **Responsible:** planning agent

## Objective and expected result

- **Objective:** move the existing `pt-BR` and `en` copy into domain-first bilingual modules without changing any key, value, locale, or route contract.
- **Observable result:** later UI work can change one product domain without editing a locale-wide copy file, while every existing `getDictionary` consumer receives the same flat typed dictionary.

## Strategy

Characterize the current exported dictionaries, then partition copy by stable product concern: shared authentication/language, provider connection, design system, administration, products, payment links, checkout policy, storefront, checkout, and orders. Each domain owns its paired `pt-BR` and `en` records so bilingual changes share one bounded write lane.

Keep `getDictionary`, `en`, and `ptBR` as server-only compatibility/assembly facades. Compose domains deterministically, preserve literal inference, and make duplicate keys, per-domain locale drift, and final assembled parity fail in focused tests. This is a structural refactor only: application consumers, copy, locale negotiation, and unprefixed routing remain unchanged.

## Affected areas

- `src/i18n/dictionaries/**` and `src/i18n/dictionaries.ts` — domain modules and stable composition facades.
- `src/i18n/dictionaries.test.ts` and `src/i18n/locales.test.ts` — structural, parity, and closed-locale regression contracts.
- `AGENTS.md` — review the `src/i18n/` ownership statement at closeout; update only if responsibility or invariants changed.

## Execution fronts

## Order and parallelism

1. **Wave 1:** F01 with one executor.
2. **Integration:** run focused dictionary/locale checks, then the aggregate database-free gate.

## Risks and abort conditions

- **Risk:** object-spread composition can silently overwrite a duplicate key — mitigation: assert disjoint domain key sets before asserting assembled locale parity.
- **Risk:** refactoring can widen literal types or break direct compatibility imports — mitigation: retain typed assembly exports and cover them through typecheck and existing consumers.
- **Abort if:** preserving current copy or export shape requires a key/value or locale behavior change — report `BLOCKED` and return to 002 with the conflicting consumer evidence.
