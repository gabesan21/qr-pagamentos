# 3.1.1-design-catalog-schema-and-admin-ui — Verification

## Summary verdict

**needs-fix**

The implementation matches the approved plan and spec for the catalog schema, service, routes, admin UI, and i18n, and the aggregate application gate passes. However, `pnpm db:contract-check` fails because the migration-history contract in `pop/scripts/db-contract-check.mjs` was not updated for the new migration. Per `prisma/AGENTS.md`, this check must pass after schema/migration changes.

## Findings

### bloqueante

1. **`pop/scripts/db-contract-check.mjs` is out of sync with the new migration.**
   - `pnpm db:contract-check` fails at line 42 with `Migration history name/count changed` because the hardcoded `migrationDirectories` array does not include `20260720230000_nautt_catalog`.
   - `prisma/AGENTS.md` mandates running `pnpm db:contract-check` after changing this subtree. The sibling `pop/scripts/db-test.mjs` was correctly updated, so this omission is a contract-check gap, not an intentional skip.
   - **Required fix:** append `20260720230000_nautt_catalog` to the `migrationDirectories` assertion; optionally add schema-model assertions for `CatalogCurrencyPair` and `CatalogPaymentMethod` to keep the contract representative.

### sugestão

1. **`src/auth/nautt-catalog.ts` uses a relative import for the database client.**
   - `import { getDatabaseClient } from "../db/client";` works but is inconsistent with the rest of the codebase, which uses the `@/db/client` alias. Consider aligning for consistency.

2. **Duplicate UUID collisions surface only as a generic create error.**
   - The unique constraints on `(currency_uuid, exchange_currency_uuid)` and `payment_method_uuid` are correct and tested at the SQL layer, but a duplicate-key Prisma error is indistinguishable from other failures in the route redirect (`?error=catalog-create-failed`). Consider adding a specific duplicate-key dictionary message if the product wants operators to distinguish bad UUIDs from already-cataloged UUIDs.

3. **The `CatalogInput` union type is declared but unused.**
   - It can be removed to avoid dead code, or it can be adopted by the service methods if future refactors want a single normalized input type.

### nit

1. **Several JSX lines in `src/app/admin/admin-surface.tsx` exceed a readable length.**
   - The `CardHeader`, definition-list, and form-action lines are long. The linter does not enforce wrapping, so this is purely stylistic.
