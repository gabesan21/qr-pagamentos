# Approval — [[3.3.2-public-link-resolution]]

## Round 1 — 2026-07-20

### Decision brief

- **Delivery:** An unlocalized, sessionless `GET /api/payment-links/[identifier]` resolver for active payment links.
- **Boundary:** Exact case-sensitive 24-character URL-safe identifier; closed `Accept-Language` locale selection; exact redacted product and currency-pair UUID DTO; uniform empty `404` for unavailable links.
- **HTTP/security:** Every outcome is `no-store`; no cookies, session/principal, preference, dictionary, writes, checkout/order/consumption behavior, or provider call is permitted.
- **Topology:** F01 owns the redacted read model and its policy tests; F02 consumes its finalized DTO at the HTTP boundary, sequentially.
- **Plan:** [[3.3.2-public-link-resolution.plan]] — *follow for the complete execution brief*.

### Resposta do revisor (yolo)

aprovado por revisor independente (yolo) — 2026-07-20

The approved contract is precise and proportionate. F01 rejects every spelling
outside the persisted URL-safe 24-character grammar before lookup, evaluates
active/expiry/product availability in one minimal read, and collapses all
unavailable states to `null`. Its explicit projection permits only localized
public product text, the untouched decimal-string price, and the configured
currency UUID pair—without IDs, administrative data, link metadata, or other
provider fields.

F02 restricts request input to the path identifier and `Accept-Language`, forces
dynamic evaluation, and maps only the resolver DTO or `null` to the exact JSON
`200` or empty `404`, each with `Cache-Control: no-store`. The sequential
ownership prevents route/resolver drift, and AC1–AC5 provide focused and
aggregate re-run evidence for identifier handling, uniform unavailability,
redaction, locale behavior, absence of session/write/provider scope, and the
final production gate. The proposed public-resolution section of the linked
catalog spec is approved with this round.
