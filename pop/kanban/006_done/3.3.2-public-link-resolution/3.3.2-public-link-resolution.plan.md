# 3.3.2-public-link-resolution — Plan

## Objective and expected result

Expose one unlocalized, sessionless JSON read at `GET /api/payment-links/[identifier]`. A valid current link returns only the localized public product and configured currency UUID pair; every unavailable spelling or state returns the same empty `404`. All responses are `Cache-Control: no-store`.

The public body is exactly `{ product: { title, description, price }, currencyPair: { currencyUuid, exchangeCurrencyUuid } }`. Product text follows the closed `pt-BR`/`en` `Accept-Language` negotiation, while `price` remains the unformatted canonical decimal string. The two pair UUIDs are the narrow exception to provider-field redaction because later checkout quoting requires them.

## Strategy

Create a small server-only resolver distinct from the protected payment-link service. It accepts an identifier and a closed `SupportedLocale`, canonical-parses the existing 24-character URL-safe, case-sensitive token without transforming it, obtains the current time through an injectable dependency, and performs one explicit redacted read. The database predicate accepts only an active link with no expiry or an expiry strictly later than that read time; it joins an active product and the configured pair. A missing row, malformed token, inactive/expired link, or inactive/missing product collapses to `null`. The pair is selected by its recorded relation without a current-active predicate, preserving the catalog promise that pair deactivation does not break existing links.

The route reads only `params.identifier` and `Accept-Language`; it uses no cookie, session, principal, locale preference, dictionary, query value, or request body. It maps `null` to an empty `404` and the exact DTO to JSON `200`, always with `no-store` and forced dynamic request evaluation. It adds neither a public checkout page nor a generated public `href`; both remain later work.

## Affected areas

- `src/auth/public-payment-link.ts` and focused test — server-only token boundary, read-store port, active/expiry/product policy, locale projection, and exact redacted DTO.

## Risks and abort conditions

- **Enumeration or stale-data leak:** a broad Prisma relation/default projection, cache, or distinct status could disclose link state. Mitigate with an explicit minimal select, exact-key assertions, uniform empty `404`, `no-store`, and forced dynamic evaluation.
- **Token ambiguity:** normalizing or decoding a case-sensitive opaque identifier can resolve a value that was not supplied. Mitigate with a single exact 24-character URL-safe grammar and no transformation before lookup.
- **Session or locale leakage:** using persisted preference or cookies would make a supposedly public read stateful. Mitigate by accepting only request `Accept-Language` through `negotiateLocale`; source and route tests forbid auth/session/cookie use.
- **Scope creep:** expiry persistence, single-use consumption, public UI, checkout/order construction, or Nautt calls would create money/state behavior without its task. Mitigate with path-level review and write-free store/route tests.
- **Abort if:** the resolver cannot join the existing pair UUID fields and active product data in one redacted read, or the approved public route/response contract must differ materially. Mark `blocked` with the concrete incompatibility and return to 002; do not invent an alternate public surface.
