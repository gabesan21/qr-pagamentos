# Plano — [[4.3.1-owner-and-admin-order-views]]

- **Etapa:** 002_planning · **Responsável:** agent planejador

## Objetivo e resultado esperado

- **Objetivo:** deliver protected, read-only order visibility: an owner-scoped order list + detail surface and an administrator global order list + detail surface, backed by one server-only projection that exposes the policy-shaped customer snapshot only to the order's owner or an administrator.
- **Resultado observável:** an authenticated owner sees exactly their own payment-link orders (state, link/product, amount/currency, policy, and collected customer data) in their persisted locale; an administrator sees the same projection across all owners; no other principal can distinguish "order exists" from "order missing", and no public surface, log, or URL gains customer data.

## Estratégia

One server-only read projection in `src/orders/` is the single seam through which order data (including the customer snapshot) leaves persistence; both UI surfaces consume it, so the authorization boundary is enforced in one place and covered by one contract-test suite. The projection is closed: order identity, payment-link identifier, product title snapshot, amount/currency pair, state and lifecycle timestamps, applied checkout policy, and the exact policy-shaped `CustomerSnapshotV1` — never retry/request/capability verifiers, nonce, key material, provider credentials, or provider envelopes. Cross-owner access by an owner resolves to the same opaque "unavailable" outcome as a missing order; administrator access is global. No mutations, no capability revocation, no Prisma schema change (the persisted model from 4.1.2 already carries owner, policy, and all customer columns). Lists are bounded and deterministically ordered (most recent first); pagination beyond a bounded recent window is out of scope. UI follows the existing shells: owner views extend the authenticated owner experience, admin views extend `src/app/admin/`; both are localized through the persisted `pt-BR`/`en` preference with matching dictionary keys. Pages re-resolve the cookie principal on every read, per the existing guards.

## Áreas afetadas

- `src/orders/` — new read-projection service + focused contract tests; existing lifecycle module stays authoritative.
- `src/app/` — new owner order list/detail routes and any API surfaces they need; `owner-guard.ts` reused, not changed.
- `src/app/admin/` — new admin order list/detail surface reusing `guard.ts` and the admin shell pattern.
- `src/i18n/dictionaries/{en,pt-BR}.ts` — new view keys in both locales.
- `DESIGN.md`, root `AGENTS.md`, `src/checkout/AGENTS.md` (lateral only), specs — DOX/spec closeout for the delivered projection slice.

## Lacunas e preflight (somente se aplicável)

- **Preflight:** `pnpm install --frozen-lockfile && pnpm db:generate` → Node/pnpm pins per `.node-version`/`package.json`; disposable DB/browser runs only if the orchestrator explicitly authorizes (prior epoch tasks skipped them).

## Frentes de execução

### F01 — Order read projection and authorization boundary

- **Entrega:** server-only owner/admin order projection service with the closed DTO above, opaque cross-owner/missing equivalence, and focused contract tests proving policy-exact snapshot exposure and boundary opacity.
- **Escopo:** reads only; no lifecycle, settlement, checkout, or capability behavior changes.
- **Owns:** `src/orders/order-view*.ts` (new module + tests).
- **May read:** `src/orders/payment-link-order.ts`, `prisma/schema.prisma`, `src/auth/authorization.ts`, `src/app/owner-guard.ts`, `src/app/admin/guard.ts`.
- **Must not edit:** `prisma/` (schema, migrations, bootstrap), `src/generated/`, `src/checkout/`, `src/integrations/`, public `/pay` and `/api/payment-links/*` surfaces, dictionaries, UI.
- **Depends on:** nenhuma.
- **Entrada esperada:** persisted `PaymentLinkOrder` model and `CustomerSnapshotV1` types from 4.1.2/4.2.1 (present; verified in recon).
- **Skills:** [[.agents/skills/clean-code-change|clean-code-change]] — *use para escrever o módulo e seus testes de contrato*.
- **Critérios:** 1, 2, 3, 4.

### F02 — Owner order views (bilingual)

- **Entrega:** authenticated owner order list + detail views rendering F01's projection in the persisted locale, with opaque empty/unavailable states and owner-guard re-authorization on every read.
- **Escopo:** read-only owner presentation; no mutations, no revocation actions.
- **Owns:** new owner order routes/components under `src/app/` (e.g. `src/app/orders/…`), their tests, and the owner-facing dictionary keys in `src/i18n/dictionaries/{en,pt-BR}.ts`.
- **May read:** F01 module, `src/app/page.tsx`, `src/app/admin/product-management.tsx` (pattern), `DESIGN.md`, `src/components/ui/AGENTS.md`.
- **Must not edit:** `src/app/admin/`, admin dictionary keys, `src/checkout/`, F01 module internals, `prisma/`, generated code; no new UI primitives outside the existing inventory without updating `DESIGN.md`.
- **Depends on:** F01.
- **Entrada esperada:** F01 projection contract (DTO shape + owner-scoped methods + opacity rules).
- **Skills:** [[.agents/skills/clean-code-change|clean-code-change]] e [[.agents/skills/ui-change|ui-change]] — *use para telas, tokens, estados e chaves i18n*.
- **Critérios:** 3, 4, 5, 6, 7.

### F03 — Admin order views (global visibility)

- **Entrega:** administrator order list + detail views over F01's admin projection, integrated into the existing admin shell, re-authorizing the cookie principal and returning only documented empty `401`/`403` outcomes on any API surface.
- **Escopo:** read-only operational inspection; no account mutation, no cross-owner management path.
- **Owns:** new admin order routes/components under `src/app/admin/`, their tests, and admin-facing dictionary keys.
- **May read:** F01 module, `src/app/admin/page.tsx`, `src/app/admin/admin-surface.tsx`, `src/auth/administration.ts`, root `AGENTS.md` admin contract.
- **Must not edit:** owner views, checkout/public surfaces, `prisma/`, F01 internals, admin mutation routes' behavior.
- **Depends on:** F01 (e F02 somente por posse de dicionário — executar após F02).
- **Entrada esperada:** F01 projection contract + F02 dictionary conventions.
- **Skills:** [[.agents/skills/clean-code-change|clean-code-change]] e [[.agents/skills/ui-change|ui-change]] — *use para a superfície admin e chaves i18n*.
- **Critérios:** 3, 4, 5, 6, 7.

## Riscos e condições de aborto

- **Risco:** vazamento de customer snapshot via log, erro, URL ou resposta pública — mitigação: projeção fechada em F01 + teste estático/contrato de que verifiers, nonce e snapshots não saem do módulo; 005 full (critical).
- **Risco:** visão admin virar caminho implícito de gestão cross-owner — mitigação: escopo read-only, sem POSTs nesta task.
- **Risco:** drift de dicionário pt-BR/en — mitigação: paridade coberta por `dictionaries.test.ts`.
- **Abortar se:** a projeção exigir mudança de schema/migration ou campo inexistente no `PaymentLinkOrder` persistido — sinalizar `blocked: true` com a coluna/campo faltante.
- **Abortar se:** F01 não puder garantir equivalência opaca cross-owner sem tocar `src/checkout/` ou as rotas públicas — sinalizar `blocked: true` com o conflito.
