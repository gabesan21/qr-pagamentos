# 3.2.2-product-public-read-boundary — Plan

## Objective and expected result

- **Objective:** introduce one server-only, product-scoped read boundary for later checkout code without turning products into a public route or exposing administrative data.
- **Observable result:** trusted server code can request one active product by its canonical UUID and a closed `SupportedLocale`, receiving either `null` or exactly `{ title, description, price }`. The text is the requested locale's persisted public copy; `price` is the unchanged canonical positive decimal string.

## Strategy

Add a small, dependency-injected public-product read service under `src/auth/`, separate from the admin CRUD service. It validates the trusted product identifier locally and performs one uncached Prisma lookup constrained by both `id` and `active: true`, selecting only the two bilingual title/description pairs and `price`. The service maps the closed `pt-BR`/`en` locale to one public title and description, returns `null` for malformed, missing, or inactive products, and does not return the identifier, internal name, active flag, version, timestamps, or any administrator/provider field.

`price` remains the persisted canonical ASCII decimal lexeme. The boundary does not parse, format, round, serialize through `Number`, or attach a currency: payment-link resolution will supply the currency in Phase 3.3. The caller already supplies the locale, so no dictionary lookup, dictionary key, locale-preference read, UI copy, or public locale route belongs here.

This is an application-internal server interface, not an HTTP surface: create no `src/app/**` route, page, handler, server action, client component, or checkout UI. The service takes a trusted persisted product UUID; the uniform `null` result still makes invalid, absent, and inactive records indistinguishable for the later public resolver. Each invocation reads the current active state directly; it must not cache a prior active result.

## Affected areas

- `src/auth/public-product.ts` — server-only redacted DTO, active-only store port, locale projection, UUID boundary, and Prisma adapter.
- `src/auth/public-product.test.ts` — injected-store contract tests for locale mapping, exact price preservation, redaction, active policy, and unavailable results.
- `pop/specs/catalog-and-payment-links.md` — the approval-pending durable clarification of this server-only boundary; promote it into the approved requirements at 003 and record the delivered slice at 006.
- Applicable repository `AGENTS.md` — review at closeout; no DOX update is expected unless the auth subtree gains a new local contract or its ownership changes.

## Dependencies, risks, and abort conditions

- **Dependency:** 3.2.1 is integrated at `d8c4fc3`; its product schema and canonical-price contract are the immutable input. This task adds no migration or database grant.
- **Redaction risk:** reusing `AdminProduct` or a broad Prisma default selection could leak internal fields. Mitigate with a distinct public DTO, explicit Prisma `select`, and exact-key tests.
- **Availability risk:** reading by `id` then filtering in memory can reveal or retain inactive products. Mitigate by applying `active: true` in the database predicate and collapsing malformed/missing/inactive results to `null`.
- **Locale risk:** a caller might receive the wrong language variant. Mitigate with `SupportedLocale` as the only locale type and paired `pt-BR`/`en` mapping tests; do not add a fallback or infer from headers/cookies here.
- **Exactness risk:** formatting or `Number` conversion can lose value. Mitigate with the maximum accepted 18/6 price lexeme and a six-decimal fractional value in focused tests, asserting byte-for-byte output.
- **Scope risk:** a route or UI would prematurely make this an externally reachable contract. Mitigate with a path-level diff review: only the two owned `src/auth` files may be application-code changes.
- **Abort if:** 3.2.1's persisted product fields or the closed locale set are missing/incompatible, or checkout needs a currency, payment link, authorization decision, or public HTTP response to consume this model. Mark the task `BLOCKED`/`RECON NEEDED` and return to 002; do not expand into Phase 3.3.
