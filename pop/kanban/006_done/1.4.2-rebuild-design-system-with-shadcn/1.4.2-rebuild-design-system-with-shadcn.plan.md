## Route

Preserve and sharpen `PIX ledger` as a restrained product system. Initialize the official named shadcn `nova` preset and accept it only when post-init resolution reports base `radix` plus style `nova`; use exact dependencies, IBM Plex Sans Variable bundled from a pinned Fontsource package, an OKLCH reference-to-semantic token graph, and Tailwind only as the shadcn composition layer. The accent remains teal and is reserved for action/focus; graphite neutrals and distinct feedback roles support both system-selected themes.

Add the smallest current-scope inventory: button, field/input, native select, checkbox, card, alert, badge, separator, skeleton, table, and spinner/loading composition. `/design-system` consumes these sources directly. Current legacy names become thin compatibility adapters to the same source until 1.4.3/1.4.4 migrate their consumers; they may not retain independent classes or styling.

Add deterministic component/token tests and a Playwright+axe browser harness. A single fresh-evidence runner creates a new run ID, captures 320/375/768/1440 in light/dark, records objective browser assertions, hashes all eight PNGs/results, and invalidates any review bound to an older manifest. Iterate screenshot -> independent UI review, bind `review.md` to that run ID and manifest hash, then validate the binding. Update `DESIGN.md`, DOX, and the draft design-system spec with the exact inventory and evidence contract.

Discarded: remote Google fonts (non-deterministic/offline-unsafe), page-local shadcn copies (drift), a second visual direction (unnecessary brand break), a manual theme toggle (outside scope), and DTCG/Terrazzo export (no current multi-platform consumer).

## Forks

- If named `nova` is rejected or does not resolve to base `radix` plus style `nova` -> abort and record the pinned CLI output; do not silently switch bases or invent a preset URL.
- If a requested official item is absent -> use the nearest official primitive composition shown by `shadcn search/view`, document the equivalence in the inventory, and do not use a community registry.
- If an existing consumer cannot accept a compatibility adapter without behavior change -> keep only that adapter as a deprecated wrapper over shared sources and add a removal note linked to 1.4.3 or 1.4.4.
- If Chromium needs system libraries but its bundled install succeeds -> run the Playwright-reported dependency check; if host installation would be required, abort rather than silently use a different browser.
- If a new visual value is needed -> add a reference token, semantic alias, contrast pair, and lint/test coverage before consuming it; never waive the raw-value gate.

## Abort conditions

- Exact registry/package lookup, frozen install, official shadcn initialization, or lockfile reproducibility cannot complete.
- Pinned Chromium cannot install and launch without a human/system-software action.
- Any external font request occurs, or the production build requires network font retrieval.
- Any required light/dark contrast pair cannot meet WCAG 2.2 AA without changing the approved one-accent direction.
- A required component needs a community registry, speculative future surface, new locale route, or auth/locale behavior change.
- Screenshot or axe execution cannot produce all eight responsive/theme results, or severity 2+ visual issues remain after two correction loops.
- Implementation needs a behaviorally significant change outside the approved spec/route; record it in the spec Open section and return to 002.
