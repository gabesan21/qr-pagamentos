- **Description:** Refine `PIX ledger` into a professional typography, color, layout, and state contract consumed by shadcn sources.

## Movements

- [x] (agent) Before editing, update/create the applicable shared-component DOX contract and characterize the current raw-value lint, contrast pairs, and theme selection.
  - Expected: root, legacy adapter, and new shared-component ownership are explicit; characterization fails only when a deliberate test fixture violates the contract.
  - If it fails: local decisions cannot be understood from the vertical contracts -> complete the narrow DOX contract first; do not edit components blind.
- [x] (agent) Preserve the `PIX ledger` identity and record the product scene/dials: payment administrators at a daily workstation, variance 4, motion 2, density 6, restrained single-teal-accent strategy.
  - Expected: `DESIGN.md` names one direction, physical reference, product register, banned defaults, and taste-as-audit boundary.
  - If it fails: a proposed choice resembles a generic SaaS/fintech reflex -> retain the receipt/ledger structure and remove decorative novelty rather than adding another direction.
- [x] (agent) Replace the token source with an OKLCH reference -> semantic -> component-consumption graph mapped to shadcn/Tailwind variables for light and dark system modes.
  - Expected: surfaces, text, action, feedback, focus, spacing, radius, shadow, type, motion, and layers are named; components contain no raw visual values or manual dark colors.
  - If it fails: a role has no valid contrast or is overloaded -> add a semantic role and matching pair test; never patch one component with a raw color.
- [x] (agent) Integrate pinned IBM Plex Sans Variable as a local build asset, define a fixed product type scale and tabular financial facts, and remove the stale system-stack claim.
  - Expected: production CSS/font assets are emitted locally, `document.fonts.ready` resolves, computed body family starts with IBM Plex Sans, and no remote font host is requested.
  - If it fails: package CSS produces uncontrolled weights or layout shift -> import only required variable ranges, reserve fallback metrics, and verify screenshots before proceeding.
- [x] (agent) Expand deterministic token tests for all rendered foreground/background, placeholder, border, focus, disabled, and feedback pairs in both themes; preserve the raw-value gate over generated/owned component source.
  - Expected: text pairs meet 4.5:1, large/non-text pairs meet 3:1, reference values remain confined to the token block, and negative fixtures still fail.
  - If it fails: clipped/out-of-gamut OKLCH or low contrast -> reduce chroma or separate semantic foregrounds while preserving hue/role; re-run every theme pair.
