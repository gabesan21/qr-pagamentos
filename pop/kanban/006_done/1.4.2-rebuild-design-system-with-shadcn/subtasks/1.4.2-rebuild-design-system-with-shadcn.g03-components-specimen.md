# Group g03 - Components, adapters, and specimen - [[1.4.2-rebuild-design-system-with-shadcn]]

- **Order:** 3.
- **Depends on:** g02.
- **Description:** Establish the one owned shadcn inventory and demonstrate every applicable state on `/design-system`.

## Movements

- [ ] (agent) Characterize current consumers, then review every CLI-added source against the shadcn form, composition, styling, alias, icon, and RSC rules before customization.
  - Expected: the inventory is source-owned under one shared subtree, uses semantic variables and built-in variants, and contains no guessed API or duplicate primitive.
  - If it fails: registry source conflicts with project lint/RSC -> make a minimal owned-source adaptation and test it; do not disable lint or copy page-local markup.
- [ ] (agent) Complete the current-scope inventory: button, field/input, native select, checkbox, card, alert, badge, separator, skeleton, table, and spinner/loading composition.
  - Expected: each component has stable props, native semantics, visible hover/focus/active/disabled behavior, and documented applicable/non-applicable loading/empty/error states.
  - If it fails: a component exists only for a future page -> remove it; a current state needs composition -> compose existing primitives before adding a new abstraction.
- [ ] (agent) Convert `ActionButton`, `Field`, `Panel`, and `Status` to deprecated compatibility adapters backed by the shared sources, preserving current public props and characterized markup behavior where required.
  - Expected: existing admin/home/language forms compile and behave, adapters carry no independent visual class contract, and removal targets 1.4.3/1.4.4 are documented.
  - If it fails: adapter changes a route/form contract -> preserve native attributes and server/client boundary, add characterization, or take the plan's single-adapter fork.
- [ ] (agent) Rebuild `/design-system` directly from shared sources with semantic sections for typography/tokens, actions, fields, feedback, loading, empty, and tabular content; use stable explicit IDs.
  - Expected: the specimen demonstrates every matrix state, one primary action per section, label-above-control, status text/icons beyond color, and no translated-title-derived IDs. Shared/specimen markup exposes deterministic `data-ds-section`, `data-variant`, `data-ds-hit-target`, `data-ds-status`, `data-ds-status-cue`, and `data-ds-prose` hooks solely for contract verification.
  - If it fails: the page becomes a generic card grid or decorative demo -> return to ledger/ruled hierarchy, reduce cards, and make every section prove a contract.
- [ ] (agent) Add focused server-render and component tests for variants, ARIA relationships, pending/disabled behavior, adapters, stable IDs, and specimen dictionary parity.
  - Expected: tests fail for missing labels/state semantics and pass for both `pt-BR`/`en` dictionary completeness without adding locale-prefixed routes.
  - If it fails: assertions depend on generated class ordering -> assert roles, attributes, variants, and source ownership instead of snapshots of incidental markup.

## Execution notes

- 2026-07-16 - The adapters are a sequencing bridge, not a second design system; downstream UI tasks remove them as they migrate consumers.
