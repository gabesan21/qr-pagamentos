## Forks

- If revoking the inherited table ACL prevents Prisma's settings update → inspect the emitted update columns; grant only an additional demonstrated column required by the existing model contract, never `INSERT`, `DELETE`, table-wide `UPDATE`, or a broad default.
- If migration replay shows the ACL comes from another role/public grant → revoke only that demonstrated grant in the same new migration and keep the effective privilege assertions unchanged.
- If bootstrap fails before migrations because the singleton is absent → keep the normalization behind an exact relation-existence guard; never create the table or swallow unrelated SQL errors in bootstrap.
- If repeated bootstrap execution changes ordinary-table privileges → restore the existing broad current/default grants and scope the exception only to `app.global_payment_settings` after those grants.
- If any login regression returns 500 or loses its cookie/locale contract after the ACL migration → do not modify authentication again; inspect migration scope and restore the already proven login candidate.
- If the exact Node tag changes again between the stale-index observation and the default checker rerun → repeat the display lookup once, update all three consumers atomically to the newly observed complete index, rerun the local equality assertion, record both observations, and rerun the default equality/child-resolution gate; do not weaken or remove the live comparison.
- If the initial display lookup cannot return the complete live index, or the post-refresh default checker cannot resolve a `linux/<host-arch>` child digest → abort at that step; never infer a digest from a prefix, local image ID, cached layer, or unrelated architecture.
- If the local assertion finds zero/multiple pins or a three-consumer mismatch → repair only the extraction or missed atomic replacement and rerun locally; registry access is not a substitute for deterministic consistency.

## Abort conditions

- Docker/Compose or its socket is unavailable in 004, so disposable database and real-runtime evidence cannot be captured.
- Reproduction requires a real user credential, persistent/non-disposable database, usable URL, or disclosure of a secret.
- The ACL correction requires weakening bootstrap-wide current/default privileges for ordinary tables, changing schema shape, prior migration history, application authorization, or the username/password, generic-failure, cookie, locale, or session contract.
- Effective least privilege cannot be demonstrated without granting runtime `INSERT`, `DELETE`, or unrestricted `UPDATE` on the singleton table.
- Digest convergence requires changing the Node version/tag/variant, using a floating or unpinned image, replacing the multi-platform index with a child manifest, altering Docker stages/helper semantics, or bypassing the existing live registry contract.

## Red-team

- **Attack that failed against the plan:** mocked unit tests could pass while production login still fails. The mandatory clean-clone production POST is a `re-run` gate and cannot be replaced by mocks.
- **Attack that passed → patch:** a generic `catch` could make the 500 disappear by falsely reporting invalid credentials, masking an outage and satisfying a shallow status check. The plan now requires an injected infrastructure-failure test, no invalid-credential redirect/cookie for that case, and diagnosis before behavior changes.
- **Attack that passed → patch:** capturing Compose logs could leak generated passwords or database URLs. The diagnosis is restricted to sanitized fields, the harness redaction sentinel remains mandatory, and any detected secret aborts the task and invalidates the artifact.
- **Return attack that passed → patch:** merely deleting the failing privilege assertion would make the gate green while preserving runtime `INSERT`/`DELETE`; the revision requires an immutable ACL migration plus positive allowed-operation and negative `42501` probes.
- **Return attack that passed → patch:** a blanket revoke could break Prisma's legitimate `SELECT`/column-update path; the contract now proves the exact allowed columns and reruns the real login outcome after migration replay.
- **Round-3 attack that passed → patch:** the forward migration can pass once and still be undone by the next idempotent bootstrap run. The plan now requires a relation-guarded normalization after bootstrap's ordinary grants and verifies the effective ACL after repeated bootstrap execution.
- **Round-3 attack that passed → patch:** narrowing default privileges globally would fix the singleton while breaking future ordinary tables. The broad defaults remain explicit and tested; only the named singleton receives the conditional exception.
- **Round-4 attack that passed → patch:** replacing the stale index with the observed child digest, a digest prefix, or only the Dockerfile occurrence could make one local build pass while breaking multi-architecture/helper reproducibility. The plan now requires the complete live index from the display lookup in all three consumers, records the resolved host child only after the successful default checker rerun, and reruns contract plus installer gates without changing the exact tag.
- **Round-5 attack that passed → patch:** the stale checker exits before raw child resolution, and its Dockerfile-only input cannot prove helper equality. The sequence now treats the initial complete mismatch as expected evidence, refreshes all consumers atomically, requires a registry-free three-consumer assertion, and only then accepts the default checker's successful live equality plus child marker; installer behavior has its own observable gate.
